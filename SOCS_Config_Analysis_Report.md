## 二、BRAM 容量详细分析

### 2.1 FPGA 资源规格（xcku3p-ffvb676-2-e）

```
BRAM数量: 216个
单个BRAM: 36KB (双端口，36Kbit × 32 = 4.5KB × 8 = 36KB)
总容量: 7776KB = 7.6MB
```

### 2.2 当前配置的实际存储需求

**基于验证系统生成的实际数据**:

```
【输入参数】
Lx = 512, Ly = 512
Kernel Size = 9×9 → Nx = 4, Ny = 4
nk = 10
sizeX = 17, sizeY = 17 (4×4+1)
```

#### 2.2.1 输入数据（AXI-MM传输，不占BRAM）

| 数据项 | 尺寸 | 字节数 | KB | 说明 |
|--------|------|--------|-----|------|
| **mskf** | 512×512 complex | 2,097,152 | 2048KB | Mask频域数据（Host已FFT） |
| **krns** | 10×17×17 complex | 23,120 | 22.6KB | SOCS核数组（10个核） |
| **scales** | 10 float | 40 | 0.04KB | 特征值数组 |

**结论**: 输入数据通过 AXI-MM burst 传输，**不占用 BRAM**。

#### 2.2.2 临时缓冲（放入BRAM）

| 缓冲项 | 尺寸 | 字节数 | KB | 说明 |
|--------|------|--------|-----|------|
| **tmp_conv** | 17×17 complex | 2,312 | 2.3KB | 点乘结果缓冲 |
| **tmp_image** | 17×17 float | 1,156 | 1.1KB | 加权累加缓冲 |
| **FFT缓冲** | 17×17 complex×2 | 4,624 | 4.5KB | IFFT双缓冲 |
| **总计** | - | 8,092 | **7.9KB** | **总临时缓冲** |

#### 2.2.3 BRAM利用率

```
总临时缓冲: 7.9KB
BRAM总容量: 7776KB
利用率: 0.1%
余量: 7768.1KB (充足)
```

**结论**: 当前配置下，BRAM 容量**极度充足**（利用率仅 0.1%）。
