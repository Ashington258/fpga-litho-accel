## 4. 关键优化成果

### 4.1 HLS FFT IP集成效果

**对比原始DFT实现**:

| 指标 | 原始DFT | HLS FFT IP | 改进 |
|------|---------|-----------|------|
| DSP占用 | 8,064 (442%) | 48 (3%) | **降低99%** ✅ |
| BRAM占用 | 1,366 (142%) | 406 (56%) | **降低70%** ✅ |
| FF占用 | 556,361 (128%) | 35,661 (10%) | **降低94%** ✅ |
| LUT占用 | 647,932 (298%) | 38,721 (23%) | **降低94%** ✅ |
| C仿真时间 | ~454s | ~150s | **加速3倍** ✅ |

**结论**: HLS FFT IP集成成功解决了原始DFT实现的资源超限问题。

### 4.2 Dataflow架构优化

**FFT 2D流水线架构**:
- **行FFT**: 128行并行处理
- **转置**: 流水线转置
- **列FFT**: 128列并行处理
- **Interval**: 83,330 cycles (支持连续数据流处理)

**优势**:
- 支持连续kernel处理，无需等待前一个kernel完成
- 吞吐量提升: 从1 kernel / 215,832 cycles → 1 kernel / 83,330 cycles (**2.6倍提升**)

### 4.3 存储架构优化

**AXI-MM接口** (6个独立接口):
- **gmem0/gmem1**: Mask频谱数据 (1024×1024 complex)
- **gmem2**: SOCS特征值 (nk=10)
- **gmem3/gmem4**: Kernel数据 (nk×17×17 complex)
- **gmem5**: 输出图像 (128×128 float)

**优势**:
- 独立接口避免总线竞争
- 支持DDR突发传输
- 带宽利用率高
