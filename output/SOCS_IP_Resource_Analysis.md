## 二、详细资源分析

### 2.1 BRAM使用详情（30块，13.9%）

#### BRAM分配明细：

| BRAM实例 | 用途 | 数量 | 配置 |
|---------|------|------|------|
| **tmp_conv_1_3_RAM** | 点乘结果缓冲（17×17 complex） | 8块 | 双端口RAM |
| **tmp_image_RAM** | 加权累加缓冲（17×17 float） | 2块 | 单端口RAM |
| **tmp_shifted_RAM** | fftshift缓冲（17×17 float） | 2块 | 单端口RAM |
| **m_axi_gmem接口** | AXI-Master读写缓冲 | 4块 | FIFO |
| **FFT临时缓冲** | IFFT输入/输出缓冲 | 8块 | 双端口RAM |
| **其他控制逻辑** | FIFO、寄存器文件 | 6块 | 混合配置 |
| **总计** | - | **30块** | - |

#### BRAM利用率分析：

```
使用量: 30块 / 216块 = 13.9%
剩余量: 186块 (86.1%)
可用于: DDR控制器、DMA、其他IP核
```

**结论**: BRAM使用量极低，**余量充足**，可支持：
- DDR控制器（MIG）：~20块BRAM
- AXI Interconnect：~10块BRAM
- 其他自定义逻辑：~50块BRAM

### 2.2 DSP使用详情（13个，1.4%）

#### DSP分配明细：

| DSP实例 | 功能 | 数量 | 实现 |
|---------|------|------|------|
| **faddfsub_32ns_32ns_32_8_full_dsp_1** | 浮点加法/减法 | 1个 | DSP48E2 |
| **fmul_32ns_32ns_32_5_max_dsp_1** | 浮点乘法 | 12个 | DSP48E2 |
| **总计** | - | **13个** | - |

#### DSP使用场景：

```cpp
// 1. 复数点乘（每个核）
tmp_conv[conv_idx] = mskf[msk_idx] * krns[krn_idx];
// 需要: 4次乘法 + 2次加法 = 6个DSP

// 2. 模平方计算
magnitude_sq = re * re + im * im;
// 需要: 2次乘法 + 1次加法 = 3个DSP

// 3. 加权累加
tmp_image[i] += scales[k] * magnitude_sq;
// 需要: 1次乘法 + 1次加法 = 2个DSP

// 4. 其他辅助运算
// 需要: 2个DSP（地址计算、索引等）
```

#### DSP利用率分析：

```
使用量: 13个 / 960个 = 1.4%
剩余量: 947个 (98.6%)
可用于: 
- FFT加速: ~100-200个DSP
- 更高并行度: 可扩展到64个并行核
- 其他计算密集型IP: ~300个DSP
```

**结论**: DSP利用率**极低**，有巨大优化空间：
- 可提高nk循环并行度（从当前串行改为4-8并行）
- 可集成真实FFT（需额外50-100个DSP）
- 可添加更多计算密集型功能
